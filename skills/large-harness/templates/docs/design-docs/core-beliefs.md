# Agent-First 核心信念

## 状态
- **分类**: Accepted
- **日期**: 2026-07-02
- **作者**: Large-Harness Skill

## 摘要
定义 agent-first 开发模式的核心信念，指导所有设计决策。

## 七条核心信念

### 1. 仓库即唯一事实来源
- 所有文档和规则都在仓库中
- 外部文档必须同步到仓库
- 口头决策必须记录到文档
- 仓库里找不到的事实视为运行上不存在

### 2. AGENTS.md 是路由器不是百科全书
- 保持简短，指向详细文档
- 只包含路由和不变量
- 不包含完整手册
- 深层规则放在 docs/ 中

### 3. 验证证据比自信更重要
- 必须有可运行的证据
- 测试通过比口头承诺更重要
- 验证结果必须可重现
- 没有证据的完成是假完成

### 4. 一次做好一个有边界任务
- 避免范围蔓延
- 不在当前任务中添加未计划的功能
- 发现新需求时记录到技术债跟踪表
- 保持专注，避免多任务并行

### 5. 反复出现的人类反馈升级成 harness 规则
- 从手动到自动
- 从口头约定到文件规则
- 从临时处理到标准流程
- 持续改进，减少人为干预

### 6. 清理与简化是交付的一部分
- 不是事后工作
- 代码清理是开发的一部分
- 文档简化是维护的一部分
- 持续优化，保持简洁

### 7. 仓库里找不到的事实视为运行上不存在
- 所有重要信息必须在仓库中
- 外部信息必须同步到仓库
- 口头决策必须记录到文档
- 信息缺失时主动提问补充

## 信念应用

### 设计决策时
- 优先考虑仓库内信息
- 确保决策可追溯
- 记录决策过程

### 实现代码时
- 遵循仓库内规范
- 保持代码简洁
- 确保可验证

### 文档编写时
- 保持文档更新
- 确保文档准确
- 提供可执行示例
