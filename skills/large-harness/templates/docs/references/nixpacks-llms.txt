# Nixpacks 部署参考

## 概述
本文档提供 Nixpacks 部署/打包规则的参考信息，包括 build 入口、runtime 假设、环境变量要求、常见失败特征。

## Build 入口

### 检测入口
Nixpacks 会自动检测项目的 build 入口：

1. **Node.js 项目**
   - 检测 `package.json`
   - 使用 `npm install` 安装依赖
   - 使用 `npm run build` 构建项目

2. **Python 项目**
   - 检测 `requirements.txt` 或 `pyproject.toml`
   - 使用 `pip install` 安装依赖
   - 无默认 build 命令

3. **Go 项目**
   - 检测 `go.mod`
   - 使用 `go build` 构建项目

4. **Rust 项目**
   - 检测 `Cargo.toml`
   - 使用 `cargo build` 构建项目

### 自定义 build 入口
在 `nixpacks.toml` 中配置：

```toml
[phases.setup]
nixPkgs = ["nodejs", "npm"]

[phases.install]
cmds = ["npm install"]

[phases.build]
cmds = ["npm run build"]

[start]
cmd = "npm start"
```

## Runtime 假设

### 环境假设
- **操作系统**: Linux (Debian-based)
- **架构**: amd64/arm64
- **用户**: 非 root 用户运行

### 文件系统假设
- **工作目录**: `/app`
- **只读文件系统**: 除 `/tmp` 和 `/app` 外
- **环境变量**: 通过环境变量传递配置

### 网络假设
- **出站网络**: 允许出站网络访问
- **入站网络**: 通过端口映射访问
- **端口**: 默认监听 `$PORT` 环境变量

## 环境变量要求

### 必需环境变量
- **PORT**: 服务监听端口，默认 `3000`

### 常用环境变量
- **NODE_ENV**: Node.js 环境，`development` 或 `production`
- **DATABASE_URL**: 数据库连接字符串
- **API_KEY**: API 密钥
- **SECRET_KEY**: 应用密钥

### 环境变量配置
```bash
# 设置环境变量
export PORT=3000
export NODE_ENV=production
export DATABASE_URL=postgresql://user:pass@localhost/db

# 运行应用
npm start
```

## 常见失败特征

### 1. 依赖安装失败
**症状**: `npm install` 或 `pip install` 失败
**原因**: 
- 网络问题
- 依赖版本冲突
- 权限问题

**解决方案**:
```bash
# 清除缓存
npm cache clean --force

# 使用 yarn 替代
yarn install

# 检查网络连接
ping registry.npmjs.org
```

### 2. Build 失败
**症状**: `npm run build` 或 `cargo build` 失败
**原因**:
- 代码编译错误
- 依赖缺失
- 环境变量缺失

**解决方案**:
```bash
# 检查代码错误
npm run lint

# 检查 TypeScript 类型
npm run typecheck

# 检查环境变量
echo $DATABASE_URL
```

### 3. 启动失败
**症状**: 应用启动失败或崩溃
**原因**:
- 端口被占用
- 依赖服务不可用
- 配置错误

**解决方案**:
```bash
# 检查端口占用
lsof -i :3000

# 检查依赖服务
ping database-service

# 检查配置文件
cat .env
```

### 4. 内存不足
**症状**: 应用被 OOM Kill
**原因**:
- 内存泄漏
- 配置不当
- 负载过高

**解决方案**:
```bash
# 增加内存限制
--memory 1g

# 检查内存使用
top

# 优化代码
npm run profile
```

## 优化建议

### Build 优化
1. **使用多阶段 build**: 分离 build 和 runtime 阶段
2. **缓存依赖**: 使用 `--cache-from` 缓存依赖
3. **并行构建**: 使用并行构建加速

### Runtime 优化
1. **使用生产依赖**: 只安装生产依赖
2. **启用压缩**: 启用 gzip 或 brotli 压缩
3. **使用 CDN**: 静态资源使用 CDN

### 镜像优化
1. **使用小基础镜像**: 使用 Alpine 或 Distroless
2. **清理缓存**: 清理不必要的缓存文件
3. **多阶段构建**: 分离 build 和 runtime 阶段

## 调试技巧

### 日志查看
```bash
# 查看构建日志
nixpacks build --verbose

# 查看运行日志
docker logs <container_id>

# 查看环境变量
env | grep PORT
```

### 远程调试
```bash
# 进入容器
docker exec -it <container_id> /bin/bash

# 检查进程
ps aux | grep node

# 检查网络
netstat -tulpn
```

### 性能分析
```bash
# 内存分析
npm run profile

# CPU 分析
node --prof app.js

# 网络分析
tcpdump -i eth0
```
