# UV Python 包管理参考

## 概述
本文档提供 UV Python 包管理的参考信息，包括 install/sync 命令、lockfile 策略、virtualenv 约定、verification 命令。

## 基本命令

### 安装依赖
```bash
# 安装单个包
uv pip install package-name

# 安装多个包
uv pip install package1 package2

# 从 requirements.txt 安装
uv pip install -r requirements.txt

# 从 pyproject.toml 安装
uv pip install -e .
```

### 同步依赖
```bash
# 同步所有依赖
uv pip sync

# 同步指定文件
uv pip sync requirements.txt

# 同步并升级
uv pip sync --upgrade
```

### 锁定依赖
```bash
# 生成 lockfile
uv pip compile requirements.in -o requirements.txt

# 升级 lockfile
uv pip compile --upgrade requirements.in -o requirements.txt

# 交叉平台锁定
uv pip compile --python-version 3.10 requirements.in -o requirements.txt
```

## Lockfile 策略

### 文件结构
```
project/
├── requirements.in      # 直接依赖
├── requirements.txt     # 锁定依赖
├── pyproject.toml       # 项目配置
└── uv.lock             # UV lockfile
```

### Lockfile 格式
```
# This file is automatically @generated by UV.
# It is not intended for manual editing.
package-name==1.2.3
another-package==4.5.6
```

### 策略原则
1. **精确版本锁定**: 使用精确版本号
2. **依赖树完整**: 包含所有传递依赖
3. **平台兼容**: 考虑跨平台兼容性
4. **安全更新**: 定期更新依赖版本

## Virtualenv 约定

### 创建 virtualenv
```bash
# 创建虚拟环境
uv venv

# 指定 Python 版本
uv venv --python 3.10

# 指定环境目录
uv venv --path .venv
```

### 激活 virtualenv
```bash
# Linux/macOS
source .venv/bin/activate

# Windows
.venv\Scripts\activate
```

### 管理 virtualenv
```bash
# 列出虚拟环境
uv venv --list

# 删除虚拟环境
rm -rf .venv

# 重新创建
uv venv --reset
```

### 约定
- **目录名**: `.venv` 或 `venv`
- **Python 版本**: 使用项目要求的 Python 版本
- **依赖隔离**: 每个项目使用独立的虚拟环境

## Verification 命令

### 依赖验证
```bash
# 检查依赖完整性
uv pip check

# 验证依赖兼容性
uv pip compile --dry-run requirements.in

# 检查安全漏洞
uv pip audit
```

### 测试验证
```bash
# 运行测试
pytest

# 运行带覆盖率的测试
pytest --cov=.

# 运行特定测试
pytest tests/test_module.py
```

### 代码质量验证
```bash
# 运行 lint
ruff check .

# 运行格式化
ruff format .

# 运行类型检查
mypy .
```

### 构建验证
```bash
# 构建包
uv build

# 验证包
twine check dist/*

# 发布到测试 PyPI
twine upload --repository testpypi dist/*
```

## 项目配置

### pyproject.toml 配置
```toml
[project]
name = "my-project"
version = "0.1.0"
description = "My awesome project"
readme = "README.md"
requires-python = ">=3.8"
dependencies = [
    "requests>=2.28.0",
    "click>=8.0.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0.0",
    "ruff>=0.1.0",
    "mypy>=1.0.0",
]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"
```

### UV 配置
```toml
[tool.uv]
dev-dependencies = [
    "pytest>=7.0.0",
    "ruff>=0.1.0",
    "mypy>=1.0.0",
]

[tool.uv.pip]
compile-args = ["--resolver=backtracking"]
```

## 最佳实践

### 依赖管理
1. **分离直接依赖和开发依赖**: 使用 `[project.optional-dependencies]`
2. **使用 lockfile**: 提交 `requirements.txt` 或 `uv.lock`
3. **定期更新**: 定期更新依赖版本
4. **安全审计**: 定期检查依赖安全漏洞

### 虚拟环境管理
1. **每个项目一个虚拟环境**: 避免依赖冲突
2. **使用 .gitignore**: 忽略虚拟环境目录
3. **使用 requirements.txt**: 记录依赖版本
4. **使用 pyproject.toml**: 定义项目元数据

### 测试和验证
1. **自动化测试**: 使用 CI/CD 自动运行测试
2. **代码质量检查**: 使用 ruff 和 mypy
3. **覆盖率报告**: 使用 pytest-cov 生成覆盖率报告
4. **性能测试**: 使用 pytest-benchmark 进行性能测试

## 常见问题

### 依赖冲突
**问题**: 安装依赖时出现版本冲突
**解决方案**:
```bash
# 使用 UV 解决冲突
uv pip compile --resolver=backtracking requirements.in

# 手动解决冲突
uv pip install package1==1.2.3
uv pip install package2==4.5.6
```

### 虚拟环境损坏
**问题**: 虚拟环境无法激活或运行
**解决方案**:
```bash
# 删除并重新创建
rm -rf .venv
uv venv

# 重新安装依赖
uv pip install -r requirements.txt
```

### 构建失败
**问题**: `uv build` 失败
**解决方案**:
```bash
# 检查构建依赖
uv pip install build

# 检查构建配置
cat pyproject.toml

# 清理构建缓存
rm -rf dist/ build/ *.egg-info/
```
