# 活跃执行计划目录

## 目的
管理所有活跃的执行计划，支持多任务并行开发。

## 文件命名规范
- **格式**: `YYYY-MM-DD-short-topic.md`
- **示例**: `2026-07-02-user-authentication.md`
- **要求**: 新到让没有聊天上下文的 agent 也能继续推进

## 当前活跃计划

### 示例计划
- **文件**: [2026-07-02-user-authentication.md](2026-07-02-user-authentication.md)
- **状态**: active
- **优先级**: high
- **负责人**: 开发团队

## 计划管理规则

### 1. 单一当前步骤
- 同一时间只有一个清晰当前步骤
- 避免多任务并行导致混乱

### 2. 实时更新
- 计划要随工作推进实时更新
- 决策改变方向时记录
- 进度变化时更新状态

### 3. 归档保留
- 完成计划归档到 `completed/` 目录
- 作为代码历史的一部分
- 供后续 agent 理解代码演进

## 计划状态流转

```
创建 → 活跃 → 完成/取消
  ↓       ↓
待定    暂停
```

### 状态说明
- **创建**: 计划刚创建，待审核
- **活跃**: 计划正在执行
- **暂停**: 计划暂时停止
- **完成**: 计划已完成
- **取消**: 计划已取消

## 计划检查清单

### 创建计划时
- [ ] 目标明确
- [ ] 范围清晰
- [ ] 验证路径可行
- [ ] 风险已识别
- [ ] 开放决策已列出

### 执行计划时
- [ ] 进度日志更新
- [ ] 风险监控
- [ ] 开放决策处理
- [ ] 验证证据收集

### 完成计划时
- [ ] 所有目标达成
- [ ] 所有验证通过
- [ ] 文档已更新
- [ ] 技术债已记录
- [ ] 计划已归档
