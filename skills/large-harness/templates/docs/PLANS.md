# 计划生命周期管理

## 何时必须创建计划

### 1. 跨会话任务
- 需要多个会话完成的任务
- 任务复杂度超过单个会话能力

### 2. 多子系统变更
- 涉及多个模块或服务
- 需要协调多个团队

### 3. 高风险操作
- 生产环境变更
- 数据库迁移
- 架构重构

### 4. 有开放决策的任务
- 需要用户确认的设计决策
- 有多种实现方案的任务

## 计划存放位置

### 活跃计划
- **位置**: `docs/exec-plans/active/`
- **命名**: `YYYY-MM-DD-short-topic.md`
- **要求**: 新到让没有聊天上下文的 agent 也能继续推进

### 已完成计划
- **位置**: `docs/exec-plans/completed/`
- **命名**: 保持原名
- **要求**: 不删除，作为仓库记忆的一部分

### 技术债跟踪
- **位置**: `docs/exec-plans/tech-debt-tracker.md`
- **内容**: 只记录真实存在、已确认、有意识延后的债务

## 计划最少包含的部分

### 1. 目标
- 明确要达成什么
- 可衡量的成功标准

### 2. 范围
- 包含什么
- 不包含什么
- 边界在哪里

### 3. 验证路径
- 如何验证完成
- 需要运行什么命令
- 需要收集什么证据

### 4. 风险
- 可能出现的问题
- 应对策略
- 回滚方案

### 5. 进度日志
- 每天的进展
- 遇到的问题
- 做出的决策

### 6. 开放决策
- 需要用户确认的问题
- 待定的设计方案
- 需要调研的技术点

## 运行规则

### 1. 单一当前步骤
- 同一时间只有一个清晰当前步骤
- 避免多任务并行导致混乱

### 2. 实时更新
- 计划要随工作推进实时更新
- 决策改变方向时记录
- 进度变化时更新状态

### 3. 归档保留
- 完成计划归档保留上下文
- 作为代码历史的一部分
- 供后续 agent 理解代码演进

## 计划模板

```markdown
# 执行计划：[计划标题]

## 基本信息
- **创建日期**: YYYY-MM-DD
- **状态**: active/completed/cancelled
- **负责人**: 人员名称
- **优先级**: high/medium/low

## 目标
[描述要达成的目标]

## 范围
### 包含
- [功能1]
- [功能2]

### 不包含
- [功能3]
- [功能4]

## 验证路径
1. [验证步骤1]
2. [验证步骤2]
3. [验证步骤3]

## 风险
### 高风险
- [风险1]: [应对策略]

### 中风险
- [风险2]: [应对策略]

## 进度日志
### YYYY-MM-DD
- [进展1]
- [进展2]
- [问题1]: [解决方案]

## 开放决策
- [决策1]: [待确认]
- [决策2]: [待调研]
```

## 技术债跟踪

### 技术债记录格式

| 日期 | 区域 | 债务 | 延后原因 | 风险 | 下次触发点 |
|------|------|------|----------|------|------------|
| 2026-07-02 | 用户认证 | 未实现 OAuth | 时间不足 | 用户无法使用第三方登录 | 下次用户要求时 |
| 2026-07-02 | 数据库 | 未优化查询 | 性能问题不明显 | 查询响应变慢 | 用户反馈性能问题时 |

### 技术债处理原则

1. **真实存在**: 只记录真实存在的债务
2. **已确认**: 已经确认需要延后
3. **有意识延后**: 有意识地决定延后处理
4. **记录完整**: 包含原因、风险和触发点
