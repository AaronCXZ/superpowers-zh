# 设计文档

## 目的
记录跨越单次聊天/sprint/reviewer 记忆的持久设计决策。

## 设计文档地图

### Accepted（已接受）
- [核心信念](design-docs/core-beliefs.md) - agent-first 核心信念

### Proposed（提议中）
- 无

### Deprecated（已弃用）
- 无

## 维护规则

1. **一个决策领域对应一份文档**
   - 保持文档小而新
   - 避免单个文档过大

2. **文档分类管理**
   - Accepted：已接受的设计决策
   - Proposed：提议中的设计决策
   - Deprecated：已弃用的设计决策

3. **版本控制**
   - 设计决策变更时更新文档
   - 保留历史版本供参考

4. **可发现性**
   - 文档标题清晰
   - 包含摘要和关键词
   - 互相链接相关文档

## 文档模板

```markdown
# [决策标题]

## 状态
- **分类**: Accepted/Proposed/Deprecated
- **日期**: YYYY-MM-DD
- **作者**: 作者名称

## 背景
描述做出这个决策的背景和原因。

## 决策
描述具体的决策内容。

## 影响
描述这个决策对系统的影响。

## 替代方案
描述考虑过的其他方案。

## 相关文档
- [相关文档1](path/to/doc1.md)
- [相关文档2](path/to/doc2.md)
```
